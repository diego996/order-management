Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

Authentication

Effettua il login e restituisce un token.

Example request:
curl --request POST \
    "http://localhost/api/login" \
    --header "Authorization: string required Credenziali in formato Basic Auth.  Example: Basic YWRtaW5AZXhhbXBsZS5jb206cGFzc3dvcmQ=  per testare con Postman admin@example.com:password" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost/api/login"
);

const headers = {
    "Authorization": "string required Credenziali in formato Basic Auth.  Example: Basic YWRtaW5AZXhhbXBsZS5jb206cGFzc3dvcmQ=  per testare con Postman admin@example.com:password",
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "POST",
    headers,
}).then(response => response.json());

Example response (200):


{
    "user": {
        "id": 1,
        "name": "Mario Rossi",
        "email": "user@example.com"
    },
    "token": "plain-text-token"
}

Example response (401):


{
    "message": "Credenziali non valide."
}

Example response (422):

    "errors": {
        "email": [
            "Email e password sono obbligatorie."
        ]
    }
}

Orders

Lista gli ordini con filtri.

requires authentication

Example request:
curl --request GET \
    --get "http://localhost/api/orders?customer_id=1&status=pending&date_from=2023-10-01&date_to=2023-10-31&per_page=15" \
    --header "Authorization: Bearer {YOUR_AUTH_KEY}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost/api/orders"
);

const params = {
    "customer_id": "1",
    "status": "pending",
    "date_from": "2023-10-01",
    "date_to": "2023-10-31",
    "per_page": "15",
};
Object.keys(params)
    .forEach(key => url.searchParams.append(key, params[key]));

const headers = {
    "Authorization": "Bearer {YOUR_AUTH_KEY}",
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());

Example response (200):


{
  "data": [array di ordini ],
  "meta": {  paginazione  }
}

Crea un nuovo ordine.

requires authentication

Example request:
curl --request POST \
    "http://localhost/api/orders" \
    --header "Authorization: Bearer {YOUR_AUTH_KEY}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"user_id\": \"consequatur\",
    \"customer_id\": 1,
    \"order_date\": \"2025-05-07T15:48:05\",
    \"order_code\": \"consequatur\",
    \"status\": \"pending\",
    \"products\": [
        {
            \"id\": 1,
            \"quantity\": 2
        }
    ]
}"

const url = new URL(
    "http://localhost/api/orders"
);

const headers = {
    "Authorization": "Bearer {YOUR_AUTH_KEY}",
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "user_id": "consequatur",
    "customer_id": 1,
    "order_date": "2025-05-07T15:48:05",
    "order_code": "consequatur",
    "status": "pending",
    "products": [
        {
            "id": 1,
            "quantity": 2
        }
    ]
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());
